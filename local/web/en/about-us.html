1. Purpose

The purpose of this project is to allow users to easily access and visualize the geographical locations of nodes and various information on a map. Users can also switch to a graph view to see areas where nodes are clustered.

2. Scope

The project displays the geographical locations of nodes on the Ethereum network based on their geolocation data (SEE FIGURE 1.1). Users can reduce the number of nodes displayed on the map using available filtering options (SEE FIGURE 1.3). If a user wants to learn more about a specific node, they can click on the node to view its details at the bottom of the map (SEE FIGURE 1.2). Some data is also displayed on the left and right sides of the website (SEE FIGURE 1.1). If a user is interested in seeing the clustering of nodes, they can press the "Graph View" button to open the graph map (SEE FIGURE 2.1). In the graph view, one or more countries can be selected, and the tree graph structure will be generated (SEE FIGURE 2.2).

3. Methodology

3.1. Data Extraction with Python

Node ID, host, port, client, and operating system information is extracted from the Etherscan website using web scraping.

3.2. ipgeolocation and API Usage

For each NODE, the host information is sent to the ipgeolocation API endpoint to obtain latitude, longitude, ISP, and country name information.

3.3. Saving Data with Python

In addition to the obtained data, the date of entry into the database and the status information are also included, and the nodes are saved by establishing connections in the Neo4j database.

3.4. Writing an API with Python

The API endpoints written with Flask are used on the front end.

3.5. Visualization as Map and Tree Graph

The website is created using HTML, CSS, and JavaScript. The JavaScript library d3.js is used for the graph view, while leaflet.js is used for the map view.

4. Conclusion

We visualized how nodes on the Ethereum network are distributed worldwide. Analysis of the map revealed that node densities are higher in certain regions, such as the United States and Europe. Users can easily visualize the geographical locations of nodes on the map and access relevant node information. By clicking on a node, users can obtain detailed information about it, and they can reduce the number of nodes displayed on the map using filtering options. Additionally, with the "Graph View" option, users can observe areas where nodes are clustered.
